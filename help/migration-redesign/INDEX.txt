KEY DESIGN DECISIONS
────────────────────────────────────────────────────────────────────────────────

✓ Schema:        2 columns (id, applied_at) vs 3 in sql-migrate
✓ SQL modes:     3 approaches (embed.FS files, Go strings, custom)
✓ Statements:    Split into individual statements (better errors)
✓ Transactions:  Per-migration control via TxDisabler interface
✓ Compatibility: Backward compatible (drop-in replacement)
✓ Performance:   30-40% fewer allocations expected
